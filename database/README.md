# PowerBlink FC — database files

**Laravel migrations are canonical.** SQL exports support phpMyAdmin fresh installs and backup review.

| Item | Purpose |
|------|--------|
| **`migrations/`** | Canonical schema — run `php artisan migrate` on each environment (after backup). |
| **`powerblink_academy.sql`** | **Primary phpMyAdmin import** — structure + showcase seed data (roles, permissions, demo academy rows, bootstrap admin). Regenerate with `php artisan db:export-powerblink-sql --fresh` on a dev MySQL instance. |
| **`schema/mysql-schema.sql`** | Structure-only SQL (generated by export command). |
| **`seed-data/powerblink/`** | Design asset manifest for `php artisan powerblink:import-design-assets`. |

---

## Fresh install (production)

1. Create utf8mb4 database in hosting panel.
2. Import **`database/powerblink_academy.sql`** via phpMyAdmin.
3. Copy `.env.example` → `.env`; set `APP_KEY`, DB credentials, Paystack keys, mail, and **strong** `BOOTSTRAP_ADMIN_PASSWORD` / `DEMO_USER_PASSWORD` (or reset passwords after import).
4. Ensure `public/asset/images/powerblink/*.jpg` is deployed (committed in Git — do not rely on manual upload).
5. `php artisan storage:link`
6. Set `ADMIN_BOOTSTRAP_ENABLED=false` and rotate admin/demo passwords before public launch.

**Bootstrap admin (included in export):** `info@powerblinkfc.com` — password is whatever was set in `BOOTSTRAP_ADMIN_PASSWORD` when the SQL dump was generated. **Never commit real production passwords**; regenerate the dump with staging credentials or reset via admin after import.

### Git deploy checklist

These must be **tracked in Git** for git-pull deploys:

| Path | Notes |
|------|--------|
| `database/powerblink_academy.sql` | phpMyAdmin import bundle |
| `database/schema/mysql-schema.sql` | Structure-only reference |
| `database/seed-data/powerblink/asset-manifest.json` | Design asset index |

`public/asset/images/powerblink/documents/` is **gitignored** — parents upload registration files on production only.

## Developer workflow

```bash
# When MySQL is not running locally (requires export passwords in env):
# PowerShell:
#   $env:BOOTSTRAP_ADMIN_PASSWORD='your-staging-bootstrap'; $env:DEMO_USER_PASSWORD='your-staging-demo'; php artisan db:export-powerblink-sql --sqlite --fresh
APP_ENV=local php artisan db:export-powerblink-sql --sqlite --fresh

# When MySQL is available (preferred on server):
php artisan db:export-powerblink-sql --fresh
```

Verify migrations sync after any import:

```bash
php artisan db:verify-migrations-sync
php artisan migrate:status   # all Ran, none Pending
php artisan test             # includes showcase density + fresh-install simulation tests
```

### Staging smoke test (before production)

1. Create an **empty** MySQL database on staging.
2. Import `database/powerblink_academy.sql` via phpMyAdmin.
3. Configure `.env` and run `php artisan migrate --force` (expect no new migrations).
4. Run `php artisan test`.
5. Log in as `info@powerblinkfc.com`, `coach@`, `parent@`, and `player@` demo accounts.
6. Spot-check admin dashboard, registration wizard, and player/parent portals on mobile widths (375px+).

### Post-export seed density check (SQLite export DB)

```bash
# After db:export-powerblink-sql --sqlite --fresh
sqlite3 database/powerblink_export.sqlite "SELECT 'players', COUNT(*) FROM players UNION ALL SELECT 'guardians', COUNT(*) FROM guardians UNION ALL SELECT 'session_attendance', COUNT(*) FROM session_attendance;"
```

Expected minimums: players ≥10, guardians ≥5, session_attendance ≥50, notifications ≥10.
